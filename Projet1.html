Contexte du Projet

L'entreprise

MyHouse est une société immobilière de 35 employés basée à Saint-Ouen, qui a déménagé dans de nouveaux locaux et souhaitait reconstruire entièrement son système d'information.

Objectifs

Concevoir une infrastructure réseau robuste, sécurisée et hautement disponible, conforme aux recommandations de l'ANSSI, avec virtualisation sous PfSense.

Besoins Techniques

Infrastructure

  • 2 contrôleurs de domaine AD
  • Cluster de fichiers DFS-R
  • Serveurs Linux Debian
  • Pare-feu Stormshield

Sécurité

  • Authentification LDAP
  • Supervision PRTG
  • Prison Fail2Ban
  • Comptes admin séparés

Spécificités

  • DMZ dédiée
  • Réseau admin isolé
  • HAProxy pour load balancing
  • Iptables pour les serveurs Webs

Architecture Réseau

Schéma de l'architecture réseau segmentée

Segmentation

  • LAN Serveurs (192.168.10.0/24) AD, DNS, DHCP, DFS-R
  • LAN Commercial (192.168.20.0/24) Postes clients
  • LAN Direction (192.168.30.0/24) Postes clients
  • LAN DSI (192.168.40.0/24) Postes clients
  • DMZ Interne (172.16.0.0/24) Serveurs Web
  • DMZ Externe (172.30.0.0/24) Serveurs HAProxy
  • LAN Admin (192.168.100.0/24) Accès sécurisé, NAT en 192.168.101.0/24

Sécurité

  • Pare-feu Stormshield avec règles strictes
  • Isolation réseau avancée
  • DMZ protégeant les services sensibles
  • Accès admin via NAT dédié
  • Journalisation centralisée
  • Certificats TLS pour tous les services
  • Mise à jour automatique des correctifs de sécurité
  • Contrôle d'intégrité des fichiers système
  • Redondance des équipements critiques
